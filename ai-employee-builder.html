Define a job to begin.
01 / The job

Give it a result, not a role.

"Marketing agent" is vague. "Find the top three customer objections every Monday and update the messaging brief" is a job.

Output: agent charter

Use an action, a result, a frequency, and a destination.

02 / The company brain

Give it the context your best employee knows.

The model is rented. Your context is proprietary. This is the layer that turns a generic assistant into your employee.

Output: context architecture
Context sources available
The generated plan separates stable context, which changes rarely, from live context, which the agent refreshes on every run.
03 / The workflow

Teach the job as a repeatable skillchain.

An agent needs a loop: observe, decide, act, verify, record. Add the real sequence your employee should follow.

Output: execution graph
04 / Tools and automation

Connect the hands, not only the brain.

Reading and writing tools make the employee useful. A trigger and an automation layer make it show up without being asked.

Output: tool and automation map
Tools the employee can use
The plan will recommend least-privilege access. Full autonomy does not require unlimited permissions.
05 / Autonomy and control

Decide what it can do without you.

Real agents act. Good agents also know when to stop, ask, retry, and leave an audit trail.

Output: approval policy
Autonomy level

Low-risk actions

Reading, analysis, drafts, internal updates.

Medium-risk actions

Changing records, messaging teams, moving work.

High-risk actions

Publishing, spending, deleting, legal or customer promises.

06 / Test and improve

Make it earn autonomy.

An AI employee starts supervised. It earns a wider job through clean runs, measurable quality, and corrections written back into the playbook.

Output: test and 30-day ramp
Never fix only the output. Fix the instruction, tool, context, or guardrail that allowed the error.
